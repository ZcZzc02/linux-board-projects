# 烧录流程

烧录会覆盖开发板存储内容，属于高风险操作。执行前必须确认设备节点、镜像路径和板卡状态。

## 风险提示

危险命令示例：

- `dd`
- 格式化磁盘
- 覆盖 eMMC、SD 卡、U 盘或板载 Flash
- 使用厂商烧录工具写入镜像

执行这类命令前必须先说明：

1. 要写入哪个设备。
2. 要写入哪个镜像。
3. 会覆盖哪些数据。
4. 如何确认设备没有选错。

## 统一烧录入口

```bash
./scripts/flash.sh
```

当前 `scripts/flash.sh` 默认不会执行烧录。必须显式设置：

```bash
FLASH_CMD="真实烧录命令" I_UNDERSTAND_FLASH_RISK=1 ./scripts/flash.sh
```

真实烧录流程确认后，再把命令固定到文档中。

## 常见错误

| 错误现象 | 可能原因 | 解决方法 |
| --- | --- | --- |
| 识别不到设备 | USB 线、驱动或烧录模式不正确 | 更换数据线并重新进入烧录模式 |
| 烧录中断 | 供电不足或连接不稳定 | 使用稳定电源和短数据线 |
| 写错设备 | 设备节点确认错误 | 烧录前重新核对 `lsblk` 输出 |
